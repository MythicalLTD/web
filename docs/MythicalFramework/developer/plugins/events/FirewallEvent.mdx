# FirewallEvent

Name: `firewall.`

Description: `Firewall events are events that are fired when a firewall action happens.`

Since: `1.0.0`

## Events

| Event Name                 | Description                                                            | Since |
|----------------------------|------------------------------------------------------------------------|-------|
| [onFirewallCheck][1]       | This event is fired when the firewall is checking an IP.         | 1.0.0 |
| [onFirewallAssignOwner][2] | This event is fired when the firewall is unblocking an IP.       | 1.0.0 |
| [onFirewallRemoveOwner][3] | This event is fired when the firewall is removing an IP      | 1.0.0 |
| [onFirewallUpdateRule][4]  | This event is fired when the firewall is updating a rule.      | 1.0.0 |

[1]: #onfirewallcheck
[2]: #onFirewallAssignOwner
[3]: #onfirewallremoveowner
[4]: #onfirewallupdaterule


### `onFirewallCheck`

FullName: `firewall.onFirewallCheck`

Description: `This event is fired when the firewall is checking an IP.`

Since: `1.0.0`

Args:
- `$ip` - The IP that is getting checked

Run:
- After the IP is checked this event is fired!

FullName: `firewall.onFirewallUpdateRule`

Description: `This event is fired when the firewall is updating a rule.`

Since: `1.0.0`

Args:
- `$ip` - The rule that is getting updated
- `$rule` - The new rule that is getting set

Run:
- After the rule is updated this event is fired!
