## 🏗️ Architecture Overview

### Backend (PHP, Laravel)
- **Chat**: Represent core data entities such as servers, users, nodes, databases, backups, permissions, and roles.
- **Controllers**: Define RESTful API endpoints for both admin and user panels, as well as the Wings daemon.
- **Middleware**: Handle authentication, admin access, and server access for API requests.
- **Plugins**: Provide a plugin system for extending backend functionality, including event hooks and plugin management.
- **Services**: Contain business logic and manage communication with the Wings daemon.
- **Routes**: Organize API endpoints by user type (admin, user, wings).
- **Helpers**: Utility functions for backend operations.

### Frontend (Vue 3, TypeScript)
- **Components**: Reusable UI elements, including server management, dashboard widgets, account management, and base UI (shacdn-vue).
- **Pages**: Route-based components for admin, authentication, and user dashboard sections.
- **Composables**: Vue composition functions for real-time updates, theme management, and navigation logic.
- **Stores**: Pinia-based state management for session, dashboard, and settings.
- **Services**: API communication and plugin management.
- **Types**: TypeScript definitions for servers, admin, and plugins.
- **Router**: Vue Router configuration for navigation.

### Key Features
- **Plugin System**: Extensible backend and frontend plugin architecture.
- **Multi-tenancy**: Support for multiple server nodes and users.
- **Real-time Updates**: WebSocket integration for live server and dashboard updates.
- **Security**: Role-based permissions, authentication middleware, and secure API endpoints.
- **Containerization**: Docker-based development and production environments.
